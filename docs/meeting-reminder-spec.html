Meeting Reminder

Schlanke macOS-Menüleisten-App. Vollbild-Overlay vor Kalender-Events. Ein-Klick-Beitritt zu MS Teams. Null CPU-Last zwischen Meetings.

✓ Scope

Must-Have

  • ✓ Vollbild-Overlay-Alert vor Kalender-Events
  • ✓ MS Teams Join-Link erkennen und per Klick öffnen
  • ✓ Kalender-Auswahl (welche Kalender überwacht werden)
  • ✓ Konfigurierbare Vorlaufzeit (Standard: 1 Minute)
  • ✓ Snooze-Funktion (1 Minute)
  • ✓ Menüleisten-App (kein Dock-Icon)
  • ✓ Events ohne Teams-Link: Ort anzeigen, Hinweis "Kein Einwahllink"
  • ✓ Option "Nur Online-Meetings"
  • ✓ Sound konfigurierbar (Standard: aus)

Nicht im Scope

  • × Andere Dienste (Zoom, Google Meet, WebEx)
  • × iOS / watchOS / iPad
  • × Eigene Kalender-Verwaltung
  • × Cloud-Sync

⏱ Scheduling

Kein Polling. Die App setzt genau einen Timer auf den nächsten Alert-Zeitpunkt und schläft dazwischen komplett.

App-Start
Kalender-Berechtigung anfragen, Events laden
Nächstes Event berechnen
Relevante Events filtern, frühestes finden
Timer setzen
EventStart − Vorlaufzeit → App schläft
Timer feuert
Vollbild-Overlay anzeigen
Nächsten Timer berechnen
Cycle wiederholt sich

Kalender-Änderungen

EKEventStoreChangedNotification abonnieren. Feuert bei neuen, geänderten oder gelöschten Events. Bei Notification: Events neu laden, Timer neu berechnen.

Spontane Meetings? Werden sofort erkannt. Exchange/iCloud synct → EventKit feuert Notification → Timer wird neu berechnet.

🔗 Teams-Erkennung

Regex auf drei Event-Felder. Priorität: url → location → notes.

https://teams\.microsoft\.com/l/meetup-join/[^\s"<>]+

Erster Treffer wird verwendet. Events ohne Treffer werden als "Kein Einwahllink" markiert.

Kalender laden
Gespeicherte Auswahl aus UserDefaults anwenden
Notification abonnieren
EKEventStoreChangedNotification
Timer setzen
Nächstes relevantes Event berechnen

Event ist "relevant" wenn

  • ✓ In einem aktivierten Kalender
  • ✓ In der Zukunft oder max 5 Min nach Start
  • ✓ Bei "Nur Online": hat Teams-Link
  • ✓ Nicht bereits dismissed
Dismissed-Set: In-Memory Set<String>. Wird NICHT persistiert. Nach App-Neustart werden laufende Events erneut angezeigt — damit man nach einem Reboot kein Meeting verpasst.

🔔 Snooze

  • ✓ "Später erinnern" → neuer Timer in 1 Minute
  • ✓ Verfügbar solange Event noch relevant (max 5 Min nach Start)
  • ✓ Event > 5 Min: kein erneuter Alert, auto-dismissed
  • ✓ Max Snooze-Kette: implizit durch 5-Min-Fenster begrenzt
